## 简介

学习目标：

- 了解 Windows 和 Linux 镜像备份方式
- 了解如何将 Linux 还原到 u 盘

## Linux dd 命令镜像备份和挂载

> 环境：Windows11
> Linux 镜像：Ubuntu 24.04

任务：首先就是用 dd 命令进行镜像备份，克隆到 u 盘里，然后挂载 u 盘提取里面的文件
dd 命令参数

- if=\[源文件\]
- of=\[目标文件\]
- status=progress 显示进度条
- bs=4M 意思是每一次传输 4M 的数据，任意都行
- conv=fsync 确保所有数据都写到物理盘上，不留缓存

1. 首先我们先用 fdisk 探测一下当前系统硬盘结构, /dev/sda 是系统盘，我们就备份这个盘，当然使用 lsblk 也是可以的
   ![Pasted%20image%2020260305093725](https://img.eval.moe/Pasted%20image%2020260305093725.png)
   ![Pasted%20image%2020260305093843](https://img.eval.moe/Pasted%20image%2020260305093843.png)
2. 镜像备份，运行完之后当前目录就会有一整个系统盘的备份 sys.bak
   ```bash
   dd if=/dev/sda of=sys.bak bs=10M status=progress conv=fsync
   ```
3. 插入 u 盘，然后将系统镜像烧录到 u 盘里

   > 如果识别不到 u 盘的话，把这里改为 3.1 即可
   > ![Pasted%20image%2020260305105530](https://img.eval.moe/Pasted%20image%2020260305105530.png)

   ![Pasted%20image%2020260305094527](https://img.eval.moe/Pasted%20image%2020260305094527.png)

4. 找到 u 盘
   ![Pasted%20image%2020260305095910](https://img.eval.moe/Pasted%20image%2020260305095910.png)
5. 将系统镜像烧录到 u 盘

```bash
sudo dd if=/home/user/backup_system.img of=/dev/sdb bs=4M status=progress conv=fsync
```

- **conv=fsync：** 这个很重要！它强制要求数据物理性地写入 U 盘，而不是留在缓存里，如果没有这个参数的话，烧录完之后要执行 sync 命令

6. 烧录完的系统大概是这样的，这里很明显 sdb1 大概率是 boot 引导分区，sdb2 不知道，sdb3 是老大，是我们的系统盘
   ![Pasted%20image%2020260305105932](https://img.eval.moe/Pasted%20image%2020260305105932.png)
7. 挂载系统盘，并查看里面的文件

```
mkdir -p /mnt/sdb/3
mount /dev/sdb3 /mnt/sdb/3
```

![Pasted%20image%2020260305110914](https://img.eval.moe/Pasted%20image%2020260305110914.png)

## Linux 备份镜像还原成系统

> 环境：Windows 11
> 工具: qemu-img
> Linux 镜像：Ubuntu

任务：将上一章用 dd 备份的数据还原成 ISO 格式，然后使用虚拟机打开该镜像

1. 下载 qemu，qumu 是一款开源的镜像转换工具，下载地址：[QEMU for Windows – Installers (64 bit)](https://qemu.weilnetz.de/w64/)
2. 配置环境变量，qemu 默认路径为：C\:\\Program Files\\qemu，输入以下命令查看系统变量是否配置成功

```powershell
C:\Users\user>qemu-img --help
qemu-img version 10.2.0 (v10.2.0-12105-g0f12d445bd)
Copyright (c) 2003-2025 Fabrice Bellard and the QEMU Project developers
QEMU disk image utility.  Usage:
...
```

3. 然后准备好一个 u 盘，这个 u 盘刚刚从 Ubuntu 中 dd 中做完镜像备份，u 盘插入的时候我们是看不见盘符的，但是我们可以在磁盘管理中看到它的存在
   ![Pasted%20image%2020260305171411](https://img.eval.moe/Pasted%20image%2020260305171411.png)
4. \*查看一下 u 盘情况，首先以管理员身份打开 powershell，并查看我们的盘的序号是多少，这里能看到是 1

```powershell
PS C:\WINDOWS\system32> Get-physicaldisk

Number FriendlyName             SerialNumber         MediaType CanPool OperationalStatus HealthStatus Usage        Size

1      WD Elements 2620         WX42A40CY0NR         HDD       True    OK                Healthy      Auto-Select ...GB
```

5. 查看 u 盘是否设置了只读，如果设置了只读的话，得改一下，主要看 Read\-only 是否为 No

```powershell
command:
diskpart
select disk 1
attribute disk
attributes disk clear readonly(如果 Read-only 为 Yes)
```

6. 用 qemu-img 将 u 盘数据转换为 vmdk，转换的过程会很慢，要等很久~

```powershell
#-p 显示进度条
#-O 转换后的格式
# \\.\PhysicalDrive1 u 盘在 Windows 里的逻辑名称
qemu-img convert -p -f raw -O vmdk \\.\PhysicalDrive1 ubuntu.vmdk
```

7. VMware 创建一个空的操作系统，系统选择和镜像里一样的，我的是 Ubuntu x64

8. 把自带的磁盘删了，选择我们刚刚转换好的磁盘

9. 开机，还原成功

## Linux 备份镜像还原成系统(第二种方法)

1. 我们如果要对系统进行镜像的话，系统的内存大概率是不够的，所以我们先给虚拟机加个硬盘
   ![fe404f37-726c-4811-a4a0-055fae3eb22e](https://img.eval.moe/fe404f37-726c-4811-a4a0-055fae3eb22e.png)
2. 启动虚拟机，然后格式化硬盘，并

```bash
# 查看大小为 40G 且没有挂在的设备，我这里是 /dev/sdb
lsblk -p
# 将硬盘格式化为 ext4 的格式
mkfs.ext4 /dev/sdb
# 创建文件夹并挂载
mkdir /mnt/new_disk
mount /dev/sdb /mnt/new_disk
```

3. 用 dd 备份系统镜像
   ![845be922-67f2-49f4-864b-cff24623b98e](https://img.eval.moe/845be922-67f2-49f4-864b-cff24623b98e.png)
4. 剩下的步骤和第一种方法一样，用 qemu-img 将 dd 文件转换为 vmdk\-\>新建虚拟机\-\>删除原有硬盘，加硬盘，选择已存在硬盘\-\>选择转换后的 vmdk 文件\-\>开机
5. 成功启动
   ![24be3d1a-01f8-4cc4-910d-a1ad76e37f41](https://img.eval.moe/24be3d1a-01f8-4cc4-910d-a1ad76e37f41.png)

## Windows 镜像备份和还原

环境：Windows 10
工具：[GitHub - threatexpert/disk2vmdk: windows 下做本地或远程的磁盘镜像的工具，可输出 vmdk · GitHub](https://github.com/threatexpert/disk2vmdk)
任务：对 Windows10 中的 E 盘进行镜像备份，格式为 vmdk，然后通过虚拟机导入还原 E 盘并提取其中的 test 文件内容

1. 打开工具，双击要备份的盘，选择要备份的分区，这里全勾上都没问题，然后选择镜像保存位置，点确定就行
   ![Pasted%20image%2020260305111826](https://img.eval.moe/Pasted%20image%2020260305111826.png)
2. 等待片刻，就会多出以下文件，1.vmdk.json 应该是元数据，不用管，重点是 1.vmdk，将 1.vmdk 从虚拟机中拷贝出来
   ![Pasted%20image%2020260305111958](https://img.eval.moe/Pasted%20image%2020260305111958.png)
3. \*修改一下文件属性，把只读给去掉，这一步很重要，不然后面会出现权限问题
   ![Pasted%20image%2020260305112159](https://img.eval.moe/Pasted%20image%2020260305112159.png)
4. 先关机，然后往虚拟机添加一块硬盘
   ![Pasted%20image%2020260305112314](https://img.eval.moe/Pasted%20image%2020260305112314.png)
5. 到这里记得选第二个选项——选择已存在的虚拟硬盘
   ![Pasted%20image%2020260305112401](https://img.eval.moe/Pasted%20image%2020260305112401.png)
6. 跳这个弹窗的话，点 Kepp Existing Format 就行
   ![Pasted%20image%2020260305112445](https://img.eval.moe/Pasted%20image%2020260305112445.png)

7. 开机，如果正常的话，在磁盘管理会多一块脱机硬盘，点击联机
   ![Pasted%20image%2020260305113752](https://img.eval.moe/Pasted%20image%2020260305113752.png)
8. 做完上述步骤后就可以在此电脑里看见自己的文件啦
   ![Pasted%20image%2020260305113830](https://img.eval.moe/Pasted%20image%2020260305113830.png)
